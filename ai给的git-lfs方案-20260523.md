## 在c代码向 `libgit2` 注册过滤器：
通过 `git_filter_register` 注册一个过滤器，将其命名为 `"lfs"`。这样一旦 `libgit2` 扫描到 `.gitattributes` 包含 `filter=lfs`，就会自动触发你的回调。

```c
#include <git2/sys/filter.h>

// 1. 检查阶段：libgit2 会自动传入匹配 `.gitattributes` 后的属性，判断是否包含期望的filter名称(例如"lfs")
static int lfs_filter_check(
    git_filter *self, void **payload, const git_filter_source *src, const char **attr_values) 
{
    // attr_values[0] 就是 .gitattributes 里 filter 属性的值
    // 如果匹配到了 "lfs"，返回 0 表示我们要接管这个文件；返回 GIT_PASSTHROUGH 表示跳过
    if (attr_values && attr_values[0] && strcmp(attr_values[0], "lfs") == 0) {
        return 0; 
    }
    return GIT_PASSTHROUGH;
}

// 2. 执行阶段：无论是 smudge (下载还原) 还是 clean (提交生成指针)，都会进到这里，
// 在这里调用java函数下载文件，日后如果有必要的话，可在这里添加任务队列，此处立即返回，
// 目前先实现成同步下载
static int lfs_filter_apply(
    git_filter *self, void **payload, git_buf *to, const git_buf *from, const git_filter_source *src) 
{
    // 判断当前是把指针变大文件（SMUDGE），还是把大文件变指针（CLEAN）
    git_filter_mode_t mode = git_filter_source_mode(src);
    // https://libgit2.org/docs/reference/main/filter/git_filter_mode_t.html
    if (mode == GIT_FILTER_SMUDGE) {
        // 如果调用java函数：
        // 把java函数传到payload，或者直接在c里写个静态的函数，调用java层固定的函数添加下载任务
        // 从 src 里可取出仓库路径，在java层可使用路径打开仓库
        // 如果使用git lfs命令：
        // 此时 `from` 内存缓冲区里是 libgit2 读到的几百字节的 LFS 指针文本。
        // 你只需要在这里通过 `popen` 调用你沙盒里的 `git-lfs smudge`：
        // 命令类似于: /data/data/your.package/files/bin/git-lfs smudge
        // 将 `from` 的内容写入该进程的 stdin，然后读取其 stdout，再写入 `to` 缓冲区。
        
    } else if (mode == GIT_FILTER_CLEAN) {
        // 此时 `from` 是大文件内容
        // 调用 /your/path/git-lfs clean，大文件流进 stdin，吐出来的指针文本写入 `to`
    }
    return 0;
}

// 3. 绑定并注册
void register_lfs_to_libgit2() {
    git_filter lfs_filter = {
        .version = GIT_FILTER_VERSION,
        .attributes = "filter=lfs", // 关键：让 libgit2 自动去帮你盯紧 .gitattributes
        .check = lfs_filter_check,
        .apply = lfs_filter_apply
    };
    // 注册为全局过滤器，优先级设置为 GIT_FILTER_DRIVER_PRIORITY（模仿核心驱动）
    git_filter_register("lfs", &lfs_filter, GIT_FILTER_DRIVER_PRIORITY);
}

```

**为什么这种方式省心？**
因为复杂的 `.gitattributes` 路径匹配、通配符支持（如 `*.psd filter=lfs`）、多层目录继承等逻辑，**全部由 `libgit2` 在底层通过 `attributes = "filter=lfs"` 自动完成了**。你的自定义代码只用安心负责临门一脚的“数据转换”。



## filter src解析：
在 `libgit2` 的自定义过滤器 API 中，`const git_filter_source *src` 是一个非常关键的**只读上下文结构体**。

它不会包含文件本身的二进制数据，而是告诉你 **“当前正在处理的这个文件叫什么名字、它属于哪个仓库、以及我们现在是在做拉取（下载）还是做提交（上传）”**。

通过调用 `libgit2` 提供的专用辅助函数（Getter），你可以从 `src` 中提取出以下核心信息：

---

### 1. 它是大文件的相对路径（最重要的值）

你在实现 LFS 过滤时，必须知道当前处理的是哪个文件（比如是 `assets/video.mp4` 还是 `docs/manual.pdf`），用来匹配你的工作区。

* **获取方法：** `git_filter_source_path(src)`
* **返回值：** 一个字符串，代表该文件相对于 Git 仓库根目录的路径。

### 2. 当前的操作模式（判断是 Smudge 还是 Clean）

你需要根据这个值来决定是应该“把指针文本变成大文件（下载）”，还是“把大文件变成指针文本（上传）”。

* **获取方法：** `git_filter_source_mode(src)`
* **返回值：** 一个枚举值，包含以下两种主要状态：
* `GIT_FILTER_SMUDGE`：**下载/检出阶段**。此时 `from` 传入的是 Git 仓库里的指针文本，你需要将真实大文件吐进 `to`。
* `GIT_FILTER_CLEAN`：**提交/暂存阶段**。此时 `from` 传入的是本地真实的大文件内容，你需要将计算好的指针文本吐进 `to`。



### 3. 当前所属的 Git 仓库指针

如果你的 Android 应用同时打开了多个 Git 仓库，这个指针能让你知道当前触发过滤的文件属于哪一个仓库，方便你去寻找对应的 `.git/lfs/` 目录。

* **获取方法：** `git_filter_source_repo(src)`
* **返回值：** 当前仓库的 `git_repository *` 指针。

### 4. 触发过滤的 Commit 或 Tree 的 OID（特定场景有用）

有时候文件并不是从本地磁盘提交，而是由于切换分支（Checkout）、或者执行归档（Archive）而触发的。

* **获取方法：** `git_filter_source_id(src)`
* **返回值：** 一个 `const git_oid *` 指针，指向当前正在被处理的底层 Git 对象的哈希值。

---

### 🛠️ 在 Android NDK (C/C++) 中如何实际编写代码？

当你的 `git_filter_apply_fn` 被回调时，你可以这样优雅地压榨 `src` 的价值：

```c
int my_lfs_filter_apply(
    git_filter *self, 
    void **payload, 
    git_buf *to, 
    const git_buf *from, 
    const git_filter_source *src) 
{
    // 1. 拿到文件路径
    const char *file_path = git_filter_source_path(src); // 例如 "main/assets/heavy_video.mp4"
    
    // 2. 拿到当前的仓库，以此推算出 .git 目录的位置
    git_repository *repo = git_filter_source_repo(src);
    const char *git_dir = git_repository_path(repo); // 例如 "/data/data/.../repo/.git/"

    // 3. 判断是该下载（Smudge）还是该切碎（Clean）
    git_filter_mode_t mode = git_filter_source_mode(src);
    
    if (mode == GIT_FILTER_SMUDGE) {
        // 逻辑 A：处理下载
        // from->ptr 里面是 LFS 指针文本，解析它！
        // 然后去 git_dir + "lfs/objects/..." 找真实文件
    } 
    else if (mode == GIT_FILTER_CLEAN) {
        // 逻辑 B：处理上传
        // from->ptr 里面是真实大文件字节流
        // 计算它的 SHA-256，生成指针写入 to
    }

    return 0; // 返回 0 表示过滤器成功执行
}

```

> ⚠️ **注意：** `src` 结构体是由 `libgit2` 底层生命周期管理的，你在回调函数里**只有只读权，千万不要去手动 free 或修改 `src` 内部的任何指针**，否则会直接引发 Android App 的底层 Segment Fault（段错误）闪退。
